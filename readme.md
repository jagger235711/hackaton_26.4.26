## 赚了么 DEMO

### 产品需求

当有一些同学炒股的时候，面对每天自己持有股票的涨跌有一种考研查分的心情--既想知道结果又不想那么直接地看到。希望以股神玩扑克的方式，一点一点的搓出结果。

### 需求论证

1. **心理防御机制**：投资者普遍存在“损失厌恶”和“鸵鸟心理”。直接面对巨大的盈亏数字往往会带来剧烈的情绪波动。通过“搓牌”式的延时揭示，为用户提供了一个心理缓冲期。
2. **多巴胺驱动的娱乐化**：将原本枯燥且高压的看盘过程“游戏化”。“揭牌”动作模仿了博弈中的高光时刻，能显著提升用户的使用频率和粘性。
3. **年轻化市场缺口**：现有的金融APP（如东方财富、同花顺）设计偏向工具化，缺乏趣味性。对于习惯于抽盲盒、玩手游的Z世代投资者，这种“盲盒式查分”具有天然吸引力。

***

### 开发方案

- **高视觉冲击力**且**功能完整**
- React
- **样式**: Tailwind CSS (极速实现霓虹暗黑风视觉)
- **动画**: **Framer Motion** (关键！用最少的代码实现丝滑的拖拽、缩放和震动效果)

#### 2. 核心视觉设计 (高冲击力)

- **暗黑背景**: 全黑背景 (#000) 配合霓虹红/绿，产生极强的金融博弈感。
- **巨型文字**: 揭开后的数字要占据屏幕 50% 以上的空间，给人视觉震撼。
- **状态渐变**: 随着揭开进度，背景颜色从黑色平滑过渡到深红或深绿。

#### 3. 简化版交互逻辑 (MVP 路径)

- **“遮罩揭露”机制**:
  - 不做复杂的 3D 弯曲，而是使用 `framer-motion` 的 `drag` 约束。
  - 用户拖拽一个带有“股神”图案的卡片覆盖层，随着偏移量增加，下方数字的 `opacity` 和 `scale` 随之变化。

**关键帧音效**:

- 拖拽时伴随“沙沙”声。
- 彻底揭开瞬间播放“金币”或“心碎”音效。

#### 4. 必须包含的功能

- 接入公开股票查询api,能够查询目标股票的涨跌
- 能够让用户导入或者手动选择目标股票

#### 5. 亮点功能 (Demo 必杀技)

- **“心跳模式”**: 在揭开过程中，手机触发轻微震动（Haptic Feedback），模拟心跳加速感。
  - 需要考虑电脑端，使用效果传达振动意图
- **“一键甩锅”**: 揭开后如果大跌，提供一个巨大的“关灯吃面”按钮，点击后屏幕瞬间全黑并显示安慰语。

### 技术架构 (Technical Architecture)

为了实现极速开发与高性能表现，采用基于 **React** 的轻量级单页应用（SPA）架构：

#### 1. 核心技术栈

- **构建工具**: `Vite` —— 相比 Next.js，在 140 分钟的 Demo 开发中，Vite 的热更新速度和配置简便性更具优势。
- **前端框架**: `React 18` —— 利用 Hooks（useState, useEffect, useContext）处理复杂的交互状态。
- **样式引擎**: `Tailwind CSS` —— 快速实现暗黑霓虹风 UI，无需编写冗长的 CSS。
- **动画引擎**: `Framer Motion` —— **核心依赖**。处理拖拽（Drag）、缩放（Scale）及揭开瞬间的震撼动效。
- **图标库**: `Lucide React` —— 快速集成高质量的金融/博弈相关图标。

#### 2. 关键组件设计

- **`ScratchCard`** **(核心组件)**:
  - 使用 `framer-motion` 的 `useMotionValue` 和 `useTransform` 实时计算拖拽距离与视觉效果（如高斯模糊 `blur`）的映射关系。
- **`StockResult`**:
  - 渲染巨型数字，并根据盈亏状态切换 Tailwind 类名（`text-red-500` / `text-green-500`）。
- **`SoundManager`**:
  - 简单的 `new Audio()` 封装，监听交互事件触发对应的音效。

#### 3. 极简目录结构 (Vite 风格)

```text
src/
├── assets/           # 音效、背景图、SVG图标
├── components/       # Card.jsx, Result.jsx, Feedback.jsx
├── hooks/            # useScratchLogic.js, useHaptic.js
├── App.jsx           # 逻辑中心，控制从“遮盖”到“揭秘”的状态转换
└── main.jsx          # 入口文件
```
